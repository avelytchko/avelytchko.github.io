Installing Pelican

Published: Wed 28 January 2015
Updated: Wed 28 January 2015
By Alexander Velytchko

In Python.

tags: pelican publishing

Pelican - это статический генератор сайтов, написанный на Python. Он похож на Ruhoh, но в отличие от него, активно развивается.

Установка

Сперва установим необходимые пакеты, если их нет. Pelican рекомендует использовать Python 2.7

$ sudo apt-get install python-pip python2.7

Воспользуемся виртуальным окружением virtualenv, чтобы не инсталлировать пакеты в систему

$ sudo pip install virtualenv
$ mkdir -p ~/virtualenvs/pelican
$ virtualenv ~/virtualenvs/pelican
$ cd ~/virtualenvs/pelican
$ source bin/activate
$ pip install pelican Markdown typogrify

Отлично, всё установлено и готово к работе. В составе Pelican'а есть замечательный визард, воспользуемся им

$ pelican-quickstart

Welcome to pelican-quickstart v3.5.0.

This script will help you create a new Pelican-based website.

Please answer the following questions so this script can generate the files
needed by Pelican.


> Where do you want to create your new web site? [.] /home/rand/Docs/avelytchko.github.io
> What will be the title of this web site? Alexander's Blog
> Who will be the author of this web site? Alexander Velytchko 
> What will be the default language of this web site? [en] ru
> Do you want to specify a URL prefix? e.g., http://example.com   (Y/n) Y
> What is your URL prefix? (see above example; no trailing slash) http://avelytchko.github.io
> Do you want to enable article pagination? (Y/n) Y
> How many articles per page do you want? [10] 
> Do you want to generate a Fabfile/Makefile to automate generation and publishing? (Y/n) Y
> Do you want an auto-reload & simpleHTTP script to assist with theme and site development? (Y/n) Y
> Do you want to upload your website using FTP? (y/N) N
> Do you want to upload your website using SSH? (y/N) N
> Do you want to upload your website using Dropbox? (y/N) N
> Do you want to upload your website using S3? (y/N) N
> Do you want to upload your website using Rackspace Cloud Files? (y/N) N
> Do you want to upload your website using GitHub Pages? (y/N) Y
> Is this your personal page (username.github.io)? (y/N) Y
Done. Your new project is available at /home/rand/Docs/avelytchko.github.io

Теперь добавим отличную Bootstrap тему к нашему будущему блогу

$ sudo apt-get install git
$ git clone https://github.com/DandyDev/pelican-bootstrap3.git

Чтобы Pelican использовал ее, нужно добавить переменную THEME в файл pelicanconf.py, указывающую на директорию, в которой находится тема

THEME = '/home/rand/Docs/avelytchko.github.io/pelican-bootstrap3'

Публикация

Приступим к написанию контента. Для языка разметки я выбрал Markdown, создайте файл с расширением .md в директории content 

$ vim /home/rand/Docs/avelytchko.github.io/content/pelican-install.md

Title: My super title
Date: 2010-12-03 10:20
Modified: 2010-12-05 19:30
Category: Python
Tags: pelican, publishing
Slug: my-super-post
Authors: Alexis Metaireau, Conan Doyle
Summary: Short version for index and feeds

This is the content of my super blog post.

Сгенерируем статичный сайт и запустим отладочный веб-сервер перед публикацией, чтобы посмотреть на результат нашей работы, доступный по адресу http://localhost:8000

$ pelican content
Done: Processed 1 article(s), 0 draft(s) and 0 page(s) in 0.57 seconds.
$ cd output
$ python -m SimpleHTTPServer

Если все прошло успешно, то пора приступить к публикации нашего блога на Github. Подразумевается, что вы уже добавили публичный ключ ~/.ssh/id_rsa.pub в свой аккаунт и создали репозиторий username.github.io

$ eval `ssh-agent`
$ ssh-add
$ cd output/
$ git config --global user.email "you@example.com"
$ git config --global user.name "Your Name"
$ git init
$ git remote add origin git@github.com:avelytchko/avelytchko.github.io.git
$ git add .
$ git commit -m 'first commit'
$ git push origin master
